# Xico 下载助手（浏览器扩展）

把浏览器当前标签里的视频/音频**真实地址**捕获出来，一键交给 **Xico** 桌面版下载。

## 为什么需要它

Xico 内置的 yt-dlp 后端能抓大多数站点，但有些视频**必须在已登录的浏览器会话里**才拿得到
真实媒体地址——最典型的就是 **X / Twitter**：其视频走 `video.twimg.com`，需要登录态/令牌，
后端在服务器侧很难直接抓。

这个扩展跑在你**已登录**的浏览器里，直接观察页面发出的媒体请求（`.m3u8` / `.mp4` / `.mpd`
/ `.ts`，尤其 `video.twimg.com`），再把**挑选出的最佳直链**通过 `xico://` 深链交给 Xico。
浏览器负责“看见”，Xico 负责“下载”。

## 工作原理（交接机制）

扩展本身**不下载任何东西**。它只做一件事：算出当前标签最该下载的 URL，然后打开一个深链：

```
xico://download?url=<对媒体或页面 URL 做 URL 编码>&kind=video|audio|image
```

macOS 收到该深链后会激活 Xico → 切到「下载器」页 → 把这个 URL 入队。
（`xico://` scheme 由 Xico 主 App 注册，**前提是你已安装 Xico**。）

popup 里“在 Xico 中下载”的按钮：
- 如果本标签**捕获到了真实媒体直链**（如那条推文的 `.mp4`）→ 交出**最佳直链**；
- 如果没捕获到 → 交出**当前页面地址**，由 Xico 内置解析器去尝试。

打开深链用的是「新建一个后台标签触发系统协议、随后马上关掉」的方式，**不会**把你正在看的
页面导航走（设计权衡详见 `chrome/popup.js` 顶部注释）。首次触发时浏览器可能弹“要打开 Xico
吗？”确认框，这是系统行为，正常预期。

## 组成

```
XicoBrowserExtension/
├── README.md                  ← 本文件
├── chrome/                    ← Chrome / Edge 的 MV3 扩展（主）
│   ├── manifest.json
│   ├── background.js          ← service worker：webRequest 观察媒体请求 + 打分排序
│   ├── content.js             ← 注入所有页面：嗅探 <video>/<source> 直链
│   ├── popup.html / .js / .css← 弹窗 UI + 深链交接
│   ├── icons/                 ← icon.svg + 生成好的 16/32/48/128 PNG
│   └── make-icons.sh          ← 由 SVG 重新生成 PNG 的脚本
└── safari/
    └── README.md              ← 转成 Safari Web Extension 的步骤与差异
```

## 安装（Chrome / Edge）

1. 打开 `chrome://extensions`（Edge 为 `edge://extensions`）。
2. 打开右上角 **开发者模式 / Developer mode**。
3. 点 **加载已解压的扩展程序 / Load unpacked**。
4. 选择本仓库的 **`chrome/` 文件夹**（不是外层 `XicoBrowserExtension/`）。
5. 扩展栏会出现紫色下载图标；建议点“固定”把它钉在工具栏。

> 需要重新生成图标时：`cd chrome && ./make-icons.sh`
> （脚本优先用 `rsvg-convert`，缺失时回退 macOS 自带的 `sips`/`qlmanage`。仓库里已附带
> 生成好的 PNG，正常无需再跑。）

## 使用

1. 打开有视频的页面（如某条 X 推文），**播放一下**让视频真正加载。
2. 点扩展图标。图标角标上的数字 = 本标签已捕获的媒体数量。
3. popup 顶部选择类型（视频/音频/图片），点 **在 Xico 中下载**（交最佳来源），
   或在下方列表里点某一条右侧的下载按钮，单独交出那一条。
4. Xico 会被唤起并把任务加入下载队列。

### X / Twitter 说明

在带视频的推文页面播放后，扩展会看到 `https://video.twimg.com/.../vid/<宽>x<高>/....mp4`
（或 `.m3u8`）请求，并把它们列出来。排序会**优先选码率/分辨率最高的 `.mp4` 完整直链**，
其次是 `.m3u8` 播放列表，**缩略图 `.jpg` 和裸 `.ts` 分段会被降权**，所以“最佳”那条通常
就是你想要的可播放视频。

## 隐私

- 为了在你点开扩展前就捕获播放器实际发出的媒体请求，Chrome 会明确提示本扩展拥有
  **“读取所有网站上的数据”**权限；启用期间，它会在普通网页标签中观察**媒体 URL**，并扫描
  `<video>/<audio>/<source>` 的直链。它不读取表单、密码、Cookie 内容或页面正文。
- 每个标签最多保留 30 条候选，只保存在**本机浏览器内存 / 会话存储**；标签关闭或导航到新页面
  就立即清除，浏览器会话结束后也不会保留。媒体 URL 可能含短期访问令牌，因此不会写入持久存储。
- popup 只能读取当前选中标签的候选；只有扩展自身 popup 能触发 `xico://download`，内容脚本不能
  借消息通道打开任意协议或读取其它标签。
- 选中的 URL 只通过本机的 `xico://` 深链交给你电脑上的 Xico App。
- **不向任何服务器发送数据**，没有分析统计、没有第三方请求、没有外部依赖。

## 已知限制

- **必须先装 Xico**：没有注册 `xico://` scheme 的话，点下载会提示打不开 Xico。
- **需要先播放**：媒体请求要在你与页面交互（播放）后才发生；一进页面还没播时可能捕获为空，
  这时可先用“下载此页面”兜底。
- **DRM / 加密流**：受 Widevine/FairPlay 等 DRM 保护的内容（如各大正版流媒体）**抓不到也
  不应抓**，本扩展不做任何 DRM 绕过。
- **blob: / MSE**：有些播放器把媒体喂给 `MediaSource`，真实分段可能是 `blob:`，无法直接交给
  外部程序下载。此时依赖网络层捕获到的 `.m3u8`/`.mp4`，或回退到页面地址。
- **`chrome://` 等特权页**：内容脚本无法注入浏览器内置页面，这些页面上扩展不工作（也无意义）。
- **Safari** 的 `webRequest` 较弱，主要靠 DOM 嗅探 + 页面地址交接，详见 `safari/README.md`。
