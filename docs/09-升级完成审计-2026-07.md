## 二、逐页对比 Sensei / iStat Menus / CleanMyMac

### 菜单栏常驻条 vs iStat Menus
- **持平或更优**：4 样式（图标+值 / 仅值 / 迷你图 / 可视化）、单色模板自适应深浅、每项独立彩色、网络永不裸 0。
- **Xico 独有的克制**：「边框只圈图形」规则——纯文字项不加框，比 iStat 的「统一背景」更干净。

### 下拉面板 vs Sensei
- **CPU**：Sensei 有频率/直方图/每核心环/进程/负载/开机时长——**Xico 现全部具备**，且每核心用迷你环、GPU 以环段并入，密度持平。
- **内存**：Sensei 有压力环/用量/分页/交换/图例——**Xico 现全部具备**，图例更完整（应用/联动/压缩/缓存/**可用** 五项对齐活动监视器口径）。
- **网络**：Sensei 有峰值/接口/单位色标——**Xico 现全部具备**，且接口行带真实 IP。
- **合并总览**：Sensei 式一屏卡片——Xico 320pt 定宽含存储/双温/风扇/电池/热状态。

### 监视页 vs Sensei / iStat
- 四大历史图平滑流动 + 悬停擦洗 + 每核心**条/环可选** + 传感器温度 + LIVE 脉冲——**持平**。

### 硬件页 vs Sensei（目标：更全）
| 维度 | Sensei | Xico | 结论 |
|---|---|---|---|
| 内存 | 容量/类型 | 容量/类型/速率/**制造商/插槽** | **更全** |
| 电池 | 循环/健康 | + **功率趋势迷你图**/温度/功率/满充设计容量 | **更全** |
| 存储 | 容量/SMART | + **TBW/剩余寿命/通电时长/读写温度**/TRIM/卷列表 | **更全** |
| 散热 | 风扇/温度 | 风扇**当前+区间条** + SoC 各簇温度（2 列传感器中心） | **持平偏优** |
| 显示 | 分辨率/刷新 | + HDR/物理尺寸/缩放 | **持平** |
| 网络 | 无独立硬件卡 | **接口清单 + IP/MAC + Wi-Fi 链路速率** | **更全** |

### 清理任务流 vs CleanMyMac
- 统一 `TaskCompletionView` 计数庆祝（喷发彩带 + 打勾弹入 + 数字 ease-out 从 0 数起），覆盖清理/粉碎/卸载/更新检查——命中 CleanMyMac 式满足感。

## 四、剩余差距 / 后续建议 —— 已全部处理（除需凭据的公证）

1. **每核心 P/E 分组的「按位」映射** — ✅ **已完成**。改为从 `IODeviceTree:/cpus` 逐 CPU 读**权威 `cluster-type`（"P"/"E"）+ `logical-cpu-id`**（`HardwareProfileService.cpuClusterTypes()`），不再猜核序；`MacInfo.coreClusters` 承载。下拉面板与监视页每核心环均按「性能核 / 能效核」真实分组标注（对齐 Sensei）；Intel/读不到时安全退化为不分组网格。真机 M1 核验：cpu0–3=E、cpu4–7=P，面板正确渲染 4 性能环 + 4 能效环。
2. **电池功率趋势迷你图** — ✅ **已补说明**。满电接通电源（功率≈0W）时如实隐藏趋势图，并显示「充放电时显示功率趋势」提示（绝不画假的平线）；充/放电时正常出现。
3. **Optimization / Maintenance 计数庆祝 + Select-All** — ✅ **已完成**。
   - 优化页「运行中的应用」：新增**全选 + 批量退出**，完成后走统一 `TaskCompletionView`（释放内存计数 + 已退出 N 个应用）。
   - 维护页：新增「**全部执行**」（免管理员任务批量运行），完成后计数庆祝（完成 N 项维护），逐项结果仍如实回写。
4. **网络接口速率共享实例** — ✅ **已完成**。硬件页、菜单栏、监视页三处各自独立 `NetworkInfoService` 实例；核验后 `env.network` 仅剩监视页网络 Tab 一个运行时消费者，基线无争用。
