# Xico 功能审计结果 + 遗留修复清单（2026-07-03）

15 个模块功能审计（每个扫描器/服务/视图都实读源码）。**结论：无 broken 模块。**

| 状态 | 模块 |
|------|------|
| works | smartScan、systemJunk/privacy/malware、spaceLens、duplicates、similarImages、uninstaller、network |
| partial | largeFiles、shredder、appUpdater、optimization、maintenance、hardware、monitor |

## 已修（build 49）
- **hardware**：CSensors.c 温度 NaN 泄漏成 `nan°C` → 正向区间过滤。
- **appUpdater**：check() 的 Task 在非主 actor 改 @Published → `Task { @MainActor in }`。
- **maintenance**：移除 `runPeriodicScripts`（`/usr/sbin/periodic` 现代 macOS 已不存在，必然静默失败）。

## 误报（已核实，不改）
- **shredder** 用 `.trash` intent：**是故意设计**（见 ShredderService 注释）。粉碎的用途就是销毁用户自有的 Documents/Downloads 文件；改 `.permanent` 会被 SafetyEngine 红线拦死，反而废掉功能。
- **optimization** 启用先改名后 bootstrap：改名=持久启用（下次登录必生效），bootstrap 只是当前会话尽力加载，失败作软警告——可接受设计。

## 遗留待办（真实但较低优先，按需再修）
- **P2 appUpdater**：网络/appcast 失败时 UI 显示"全部最新"，无法区分"无更新"与"检查失败"。需加 `checkError` 贯通 service→model→view。
- **P2 largeFiles**：标题"大文件与旧文件"但只按大小(≥100MB)检测，未用已读取的 modificationDate 做"旧文件"过滤。要么改名"大文件"，要么真正实现按时间过滤。
- **P2 monitor**：历史图(hour/day/week)用归一化值画线，但文字标签仍是当前快照的原始 bytes/sec，量纲不一致。
- **P2 duplicates**：重复组静默截断到 200，超限应设 scanWarning 提示。
- **P2 uninstaller**：选中 app 后拉取关联文件期间无 loading，闪一下空态。
- **P2 threatSignatures**：24 条签名硬编码，definitions.json 无 `threatSignatures` 字段，威胁库无法离线更新（更新通道基建已在，差数据源；先加 `[]` 打通）。
- **P3 批量**：spaceLens 聚合节点用父目录 URL(「在Finder显示」跳错)/命名不一致/无 deinit 取消；optimization 各类健壮性;maintenance killall 退出码;shredder symlink 大小不计入 freed + 缺目录树测试;hardware 温度下限 `<=0` 与 Intel 路径不对称;network publicIP 失败不冷却;duplicates 死代码 do 块。
