## 〇、一句话战略

**不在任何单一竞品的主场做单点对轰**——iStat 的监控密度、MacPaw 的病毒库都是重资产护城河——而用「**三合一整合 + 统一的精致高级设计语言 + 更准更安全更诚实的账本 + 中国区/开发者专清**」打竞品结构性无法跟进的空档。

**四个赢法支点：**

1. **整合体验（最硬，任何单品做不到）**：菜单栏发现异常 → 一键深链清理／透镜定位；透镜里看到可清理物 → 直接入清理引擎；三处删除全走**同一条红线 SafetyEngine + 同一个撤销栈**。iStat 只能看不能做、DaisyDisk 只删空间不懂垃圾、CMM/柠檬没有空间可视化——三合一是结构性碾压。
2. **更准 + 更诚实的账本（信任牌）**：物理口径统一 + APFS clone 去重 + 删后卷级实测回填 + purgeable/快照「只解释不计入」。正面打 CMM「6.1 vs 7.1 数字打架」的软肋。
3. **更安全（唯一无法被病毒库重资产反制的护城河）**：三道闸纵深防御 + 逐规则可解释 + 100% 可撤销 + **比 CMM 更克制的默认勾选纪律**。
4. **更好看 + 更本地化**：极光渐变身份贯通菜单栏/清理/透镜 + warmLuxe/jewel 温暖珠宝主题 + 11 语言 + 中国区微信 6 粒度专清 + Chromium 通用引擎。

> **诚实的现状摸底（决定了打法是"扬长补短"而非空谈）：**
> - **状态栏**：底子已不错（CG 直绘、玻璃卡片、逐项定制、真·合并项、内存压力口径同 iStat），差在「配色高级感 + 指标全面度 + 打通体验」，且有 **3 处断裂 bug**（主题联动死代码 / 网络样式等价 / 换入换出用累计值）。
> - **清理**：`SafetyEngine` 是**世界级**的（fail-closed 纵深防御、凭证目录零删除），开发者残余覆盖**已超** CMM/柠檬（29 条）。真正缺口是 **零中国区 App**（微信/QQ/钉钉/网盘）与 4 处激进默认勾。
> - **空间透镜**：扫描引擎**已是 DaisyDisk 级甚至更强**（getattrlistbulk + iCloud 云占位红线 + 硬链接去重 + 挂载点/firmlink/.nofollow 跳过 + 隐藏空间诚实口径 + 物理已分配字节）。超越点在**旭日环颜值 + 打通清理引擎（DaisyDisk 没有的杀手锏）+ clone 去重 + 快照可删**。

## 一、跨域统一设计语言：一套「精致高级含温暖系」规范

把配色从「菜单栏专用」提升为**贯穿菜单栏/清理/透镜的全局令牌体系**。原则：单一取色入口（`XColor` 门面）+ 主题驱动 + 深浅色 `dynamic()` 自适应，三个模块**零硬编码颜色**。

### 1.1 三层色彩语义（全局统一）

| 层 | 令牌 | 菜单栏用法 | 清理用法 | 透镜用法 |
|---|---|---|---|---|
| **品牌身份层** | 极光渐变 蓝`0x5478F0`→紫`0x8B6FE6`→兰`0xB873D8`；warmLuxe/jewel 暖调 | 彩色模式渐变身份 | 健康分环/一键扫描主视觉 | 中心盘底座 + 环整体呼吸 |
| **指标/家族层** | `ring[0..3]` 派生 metricCPU/Memory/GPU/Disk/Network | 逐指标互不撞色 | —（清理不用指标色） | `vividPalette` 家族色相改为从 ring 派生，与菜单栏同源 |
| **语义层** | success/`warning 0xF59E0B`/danger + idle 中性 | 告警态 | **SafetyLevel 三色**：safe=success、caution=warning、risky=danger | 「可安全清理」徽标=success；隐藏空间=idle 中性灰 |

**SafetyLevel 三色贯穿清理 UI**——让「安全度」在扫描列表、透镜徽标、收集篮里视觉一致，**安全度即配色**，这是「更安全」卖点的视觉化。

### 1.5 必需的结构补丁（6 行，让 `menuDisk/menuGPU` 生效；旧六套主题零改动）

现状 `MenuBarGlyph.swift:165` 磁盘固定 `[XColor.warning]`、`:183` GPU 固定 `[XColor.accentPink]`，与主题无关。这正是 Xico「5 指标挤 4 槽 ring」的历史派生——把这两枚「第 5/6 色」交还主题：

```swift
// ① Theme.swift — XTheme 增两个可选字段（默认 nil = 回退旧行为）
public let menuGPU: Color?
public let menuDisk: Color?
// init 增参数 menuGPU: Color? = nil, menuDisk: Color? = nil，并赋值

// ② Tokens.swift — XColor 门面（与 metricCPU 等并列，单一取色入口）
public static var menuGPU:  Color { XThemeStore.shared.current.menuGPU  ?? accentPink }
public static var menuDisk: Color { XThemeStore.shared.current.menuDisk ?? warning }

// ③ MenuBarGlyph.swift — 两处硬编码改读主题
// :165  磁盘   tint: [XColor.menuDisk]
// :183  GPU    tint: [XColor.menuGPU]
```

CPU/内存/网络已走 `metricCPU/Memory/Network` = `ring(2)/ring(1)/ring(3)`；上面两套 ring 顺序**特意排成 ring2=CPU、ring1=内存、ring3=网络**，故三指标**无需改代码**即命中色相。注册：`Theme.swift:62` 的 `all` 追加 `warmLuxe, jewel`。

### 1.6 彩色三铁律（全局，源自 WCAG 硬账）

1. **模板单色是默认地基，彩色是 opt-in**——数学事实：纯色数字无法在浅栏+深栏同时达 WCAG 文本 4.5:1（需相对亮度 L∈[0.175, 0.183] 近乎无解）。彩色只锚定「非文本图形 3:1」（WCAG 1.4.11）并尽量逼近。两套 light 版全 ≥3.5:1、dark 版全 ≥6:1。
2. **每指标「纯色相 + 图标 + 数值」三重编码**（HIG 别只靠颜色，刘海裁半仍可认）。
3. **暖不塌成一坨**：推明度别推饱和、绕开泥黄绿死区（hue 60–90° 压到 36–42°）、暖色配中性冷墨底（沿用现有 `canvasBottom #07080F`，别把背景调暖）、每套借一枚冷宝石破红绿盲。

## 二、跨域杀手锏（三模块打通，单一竞品做不到）

这是总方案的**核心价值**——每条都需要至少两个模块协同，任何单品无法复制。

- **KILLER-1｜异常发现 → 一键处置闭环**：菜单栏「磁盘/内存」面板底部两个深链按钮：「一键清理」深链 `SmartScanHub`、「深潜空间透镜」深链 `SunburstView`。菜单栏从「看数字」升级为「看→点→清/定位」。iStat 只能看、CMM 无监控入口、DaisyDisk 无清理——三边空档。
- **KILLER-2｜统一删除引擎 + 全局撤销栈**：把透镜收集篮的裸 `NSWorkspace.recycle`（`SpaceLensView.swift:281`）改走 `CleaningEngine.execute(CleaningPlan)`。收益三合一：① 透镜也有**应用内一键 Undo**（DaisyDisk 永久删除做不到）；② 删除口径统一到 `SafetyEngine.verify` + TOCTOU 复校；③ 清理页与透镜同一份报告口径。仅需 `DiskNode→CleanableItem` 一层适配。
- **KILLER-3｜诚实账本三本账**：purgeable/快照/无权限区一份服务同时喂 `SmartScanHub` 结果页与 `SpaceLens` 中心盘。透镜里快照**破例可一键删**（`tmutil deletelocalsnapshots`，二次确认独立分支），清理页「只解释不计入」。把 CMM 最大软肋做成 Xico 信任展示位（「预计 X / 实测释放 Y」）。
  > 校正：`SpaceLedger.swift` **已实现真实 purgeable**（`collect()` 读 `volumeAvailableCapacityForImportantUsage − volumeAvailableCapacity` + `tmutil listlocalsnapshots`），`SmartScanHub.swift:329` 落定后已回填。`SmartScanHub.swift:528` 的 4.6GB 常量只在 demo/海报路径，**别动**。本项主要是「验证真机三本账真实展示」。
- **KILLER-4｜可安全清理徽标**：透镜扫到「已知可清理类型」（缓存/日志/DerivedData，`SafetyEngine.permanentHomeAllowlistLower:37` 已枚举）→ 环段/图例给 success 徽标 → 点击直接入收集篮。空间可视化与智能清理在同一屏化学反应。
- **KILLER-5｜温暖高级主题 + 场景预设身份统一**：warmLuxe/jewel 珠宝暖调 + 「游戏/续航/开发者」一键场景预设，菜单栏与 App 内极光渐变身份贯通。命名即卖点（黄玉/石榴/紫晶）。打 iStat 四层配置过载 + 英文老派。

## 五、空间透镜：飞跃式超越 DaisyDisk

**结论先行**：扫描内核与删除安全闭环已达/局部超越 DaisyDisk 工程水准，**真正的飞跃点在四处**——(A) APFS clone/快照准确性根治、(B) 旭日环顶级美化 + 可用空间语境 + 环上标签、(C) 收集篮与全应用清理引擎打通（DaisyDisk 没有的整合杀手锏）、(D) 多卷 + 跨树搜索。

### 5.1 已达标（勿重复造）

硬链接去重（`countFirstSighting`）、稀疏/压缩按物理（`ATTR_FILE_ALLOCSIZE`）、不跨挂载点（`DIR_MNTSTATUS_MNTPOINT|TRIGGER`）、不跟符号链接（`O_NOFOLLOW`）、根魔法项跳过、云占位不物化（`setiopolicy_np`）、隐藏空间对账、移废纸篓可恢复、两段式收集篮 5s 倒计时。**这些是 DaisyDisk 同款正解，是资产。**

### 5.2 P0（准确性 + 竞争缺口）

- **P0-1 修 APFS clone 重复计数**（最大准确缺口）：现只对硬链接去重，对 CoW clone（`cp -c`、DerivedData、原子写）**零去重**。落地推荐**方案 A**：`BulkDirectoryReader` 的 `attrs.forkattr` 加 `ATTR_CMNEXT_PRIVATESIZE`（macOS 10.13+），节点尺寸优先 privatesize（独占块）作真值，另存 allocsize 作展示。总量 = Σprivatesize = 真实可释放，**无需全局 clone 表、零额外内存**。缺失时优雅回退 allocsize。
- **P0-2 隐藏空间拆分 + 本地快照一键删**：`hiddenSpaceNode:298` 单块灰升级为可展开：`本地快照(可删)` / `可清除(purgeable)` / `无权限读取区`。快照删除走独立 `tmutil deletelocalsnapshots` 通道，**二次确认 + `env.safety` 独立放行分支**，绝不复用普通删除路径。
- **P0-3 收集篮改走 CleaningEngine + 应用内 Undo**（KILLER-2）：`trashMany:281` 改构造 `CleaningPlan` 调 `env.cleaningEngine.execute`，庆祝页加「撤销」按钮调 `CleaningEngine.undo`。

### 5.3 P1 / P2

- **P1**：并发信号量限流（`DiskTreeScanner:125`，`activeProcessorCount*4`）、分片去重表（`fileID % 64`）、无权限区引导 FDA、**环段内嵌切向标签**（sweep≥18°且径向≥28pt 才画，字色 `readableText`）、卷根「可用空间」低饱和楔形、iCloud 就地驱逐（`ICloudEvictor`）、`⌘⌫` 键盘删除。
- **P2**：增量 mtime 缓存（加 `ATTR_CMN_MODTIME`，目录未变复用子树——**DaisyDisk 无此项，净胜**）、多卷视图 + 跨树搜索、Treemap squarified、中心盘 Liquid Glass 质感、可安全清理徽标联动。

### 5.4 计分卡 vs DaisyDisk

| 维度 | DaisyDisk | Xico 落地后 | 赢法 |
|---|---|---|---|
| **更好看** | 色轮继承 + blossom，深色刚补齐 | 色轮 + 家族色相 + 双主题已打平 | 环上标签 + 可用空间楔形 + Liquid Glass（深色本是 DaisyDisk 弱项） |

## 八、风险与红线

1. **安全（SafetyEngine 是唯一不可退让的红线）**：三道闸 + 共用红线**必须保持**；KILLER-2 让透镜也走这条链是**加固**不是绕过。CI 固化 `libraryExtraAllowedSubtrees` 与 `definitions.json` 同步。快照删除走 `tmutil` 是系统级不可移废纸篓操作，**二次确认 + 独立分支，绝不进一键批量**。risky 项（微信 `Message/*.db`、`Docker.raw`、Cookies/Login Data、语言包/通用二进制瘦身）**永不删只提示**。
2. **性能**：菜单栏采样每 2s 全量——Rules 引擎与多传感器不得加重主线程（runloop `.common` 是前置）；透镜并发需信号量限流，与菜单栏采样共享 CPU 预算。
3. **体积/口径一致性（信任裂缝）**：全局统一到物理占用（`.totalFileAllocatedSizeKey`），禁用 `.fileSize`；宣传写「**预计可释放**」删后回填真值；purgeable/快照**只解释不计入**。
4. **审核/合规**：系统缓存走特权助手需 XPC 签名与公证链完整（公证阻塞项独立于本设计）；中国区路径动态枚举 + 走签名通道下发；FDA 缺失时显式引导而非静默少算。
