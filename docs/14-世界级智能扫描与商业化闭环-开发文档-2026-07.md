## 二、对标 CleanMyMac 5 Smart Care：差距总表

|---|--------------------|-----------|----------|

## 三、新信息架构（P0）

### 3.1 侧边栏：四组 → 三组

| 组 | 之前 | 之后 |
|----|------|------|
| 清理 | 智能扫描 · 系统垃圾 · 废纸篓 | **智能扫描 · 系统垃圾** |
| 应用 | 卸载器 · 应用更新 | **卸载器 · 应用更新 · 空间透镜** |
| 文件与空间 | 空间透镜 · 大文件与旧文件 · 重复文件 · 相似图片 · 文件粉碎 | **（撤销该组）** |
| 性能与安全 | 优化 · 维护 · 威胁防护 · 磁盘测速 · 硬件 · 系统监视 | **状态栏（新）· 优化 · 维护 · 磁盘测速 · 硬件 · 系统监视** |

去向：废纸篓 / 大文件与旧文件 / 重复文件 / 相似图片 / 威胁防护 → **并入智能扫描六类目**；文件粉碎 → 智能扫描页工具入口；状态栏 → 从设置页抽出为独立页（置于「优化」上方）。

### 3.2 工程要点（侦察结论）

1. 侧边栏是数据驱动：`ModuleCatalog.all`（XicoEnvironment.swift:139-161）+ `grouped()`（:163-167）。**隐藏 ≠ 删除**：被并入的模块从目录移除侧边栏展示但保留 DetailView 路由（RootView.swift:292-317）与 `--open=` CLI 直达（与 `.privacy` 先例一致，RootView.swift:306）。
2. `ModuleScanView` 靠 `ModuleCatalog.all` 查元数据（ScanViews.swift:680）——给 `ModuleMetadata` 加 `sidebar: Bool` 标志并让 `grouped()` 过滤，比从 `all` 删行安全。
3. `categoryTint` 按**分组位置**取色（RootView.swift:136-146）：撤掉文件与空间后索引 2 = 性能与安全，会错染成紫色——调色板须同步改为 [青, 蓝, 橙]。
4. 空分组仍渲染组头（RootView.swift:114）——`grouped()` 需过滤空分组，或删除 `ModuleCategory.filesSpace` 枚举 case（未持久化，删除安全）。
5. 新「状态栏」页 = SettingsView 的 menuBarCard 整体迁出（SettingsView.swift:603-654 + MBItem :514-523 + 动态绑定 :570-601 + 预设 :708-733 + MBStyleTile 等私有预览 :1008-1148）。所有状态已外化在 `xico.mb.*` UserDefaults，MenuBarController 靠 defaults 快照差分自重建（MenuBarController.swift:59-78）——**零控制器改动**。菜单栏右键「设置…」目标（MenuBarController.swift:139）改指新页。

## 四、智能扫描中枢架构（P1）

### 4.1 六类目

| 类目 | 扫描器 | 清理意图 | 自动勾选策略 |
|------|--------|----------|--------------|
| 系统垃圾 | SystemJunkScanner + PrivacyScanner + DeepScanner（现有三件套） | .trash | safe 项自动选（现状） |
| 废纸篓 | TrashScanner（Scanners.swift:464，含外置盘 .Trashes） | **.permanent**（清空） | 全选但清理前确认 |
| 大文件与旧文件 | LargeFilesScanner（Scanners.swift:332） | .trash | **永不自动选**（个人数据） |
| 重复文件 | DuplicatesScanner（默认 ~/Downloads 根，可换） | .trash | 组内保留最优，其余可选 |
| 相似图片 | SimilarImagesScanner | .trash | 同上 |
| 威胁防护 | ThreatScanner（四层检测） | .trash | **永不自动选**（.risky 现状），清理前 launchctl bootout |

### 4.2 关键设计决策

1. **混合意图 → 新建 Hub VM**。`ModuleSessionViewModel` 单意图（ModuleSessionViewModel.swift:70），废纸篓需 .permanent 而其余 .trash——所以中枢不复用单会话 VM，新建 `SmartScanHubViewModel`：每类目独立状态机（pending/scanning/done/failed + groups + included 开关），清理按类目分批走 `env.cleaningEngine.execute`（红线、撤销、历史全部免费继承）。
2. **逐类到达（S1）**：`withTaskGroup` 每类目独立 Task，完成即发布——不等最慢者。类目内部失败不拖垮整体（FailureBox 先例，ScanViews.swift:647）。
3. **威胁 bootout 钩子**：智能扫描会话此前无 `beforeClean`（AppModel.swift:222-236 缺口）——Hub 清理威胁类目前必须 `ThreatRemediation.bootoutUserAgents`（自带 ~/Library/LaunchAgents 限定 + Label 白名单校验，ThreatRemediation.swift:34）。
4. **许可门与红线不动**：扫描免费、清理走 `allowsCommercialUse` 门（ensureLicensed 模式，ModuleSessionViewModel.swift:326）；每一次删除仍经 CleaningEngine 的 SafetyEngine verify + 符号链接 TOCTOU 复查。
5. **下钻复用**：类目 Review 页复用 `ResultGroupCard`/`ItemRowView`（SharedViews.swift:104/34）——不重写列表。
6. **文件粉碎**：不是扫描类目，作为中枢页工具入口（跳 `.shredder` 路由）。
7. **性能红线**：重扫描类目（大文件/重复/相似）此前被刻意排除在智能扫描外是因为拖慢一键体验（XicoEnvironment.swift:120-124 注释）——逐类到达架构（S1）正是解法：快类目秒出，重类目后台补齐，体验反而成为卖点。

### 4.3 诚实规则（继承全项目铁律）

- 读不到就说读不到：FDA 缺失显式标注「再授权可多扫 N 类」（权限诚实先例 ModuleSessionViewModel.swift:147-167），绝不静默少扫装作「很干净」。
- 体积数字 = `fs.allocatedSize` 实测，绝不把 purgeable 算进「可回收」（P3 进一步分账）。
- 日志/缓存类默认保守：macOS 自动轮转日志（Eclectic Light 结论），文案不鼓吹全清。

### P7 · 磁盘测速引擎 v2（对标并超越 Blackmagic Disk Speed Test）✅ 已实施（2026-07-11）

**目标**：测速数字既要**比 Blackmagic 高**（喂饱介质）又要**比它诚实**（直读介质、方法学可见），并补上它没有的专业层（RND4K IOPS/延迟、三态视频适配表、外置卷测速）。

**根因诊断（本机 M1 实测标定）**：旧引擎 读 2686 / 写 1142 MB/s vs Blackmagic 读 2951 / 写 2146。四个因子分解实验（scratchpad/benchprobe.c）：①缓冲未对齐 −31%（malloc 1196 → 16KB 对齐 1567 MB/s——Swift `[UInt8]` 即未对齐！）；②单线程 QD1 喂不饱 Apple NVMe 写（QD2 → 1958）；③F_FULLFSYNC 只占 0.01–0.46s **不是**主因（但曾被计入耗时）；④8MB 块劣于 64/128MB。竞品情报：**BMDST 根本没用 F_NOCACHE**（大内存机型读速被系统缓存抬高）、无中位数/IOPS/延迟、仪表为逐轮瞬时值。

**已实施**（Infrastructure/DiskBenchmark.swift 重写 + Domain/VideoFitReference.swift + DiskBenchmarkView 升级）：
1. **SEQ 引擎**：posix_memalign 16KB 对齐缓冲 + 64MB 块 × QD2（双线程 pread/pwrite 不相交区段）+ F_NOCACHE + F_RDAHEAD 0 + F_PREALLOCATE 连续预分配 + 不可压缩数据；瞬时速度 = 0.25s 窗口增量（非累计均值）；**F_FULLFSYNC 单独计时展示、不计入吞吐**（对齐 BMDST/CDM 口径，但比它们多一层可见的诚实）。
2. **RND4K 矩阵**：QD1 读/写（IOPS + 平均 + p99 延迟，clock 逐次计时）+ QD32 深队列饱和 IOPS（32 线程同步 pread/pwrite——macOS 无 io_uring 的正解，AmorphousDiskMark 同法）。
3. **外置卷测速**：测试文件写在所选卷（temporaryDirectory 永远在系统卷是经典错误）；NSOpenPanel 选卷 + 只读卷拒绝。
4. **视频适配表**：Apple ProRes 白皮书 2022 官方码率 + BRAW 官方公式（已验证 12K 5:1 → 573≈官方 578）；**三态判定 ✓/⚠/✗（1.2× 安全系数）**——BMDST 手册自认「勾但刚好达标 = marginal」，我们把这层含糊做成显式边缘态。
5. SLC 爆发观察（首秒 vs 持续差 >15% 点名缓存效应）+ 方法学脚注 + 历史模型向后兼容（新字段全 optional）。

**验收实测（同机同日）**：v2 读 **3326**（+24% vs 旧，+13% vs BMDST 2951）/ 写 **2652 MB/s**（+132% vs 旧，+24% vs BMDST 2146）；RND4K QD1 读 14.9K IOPS @66µs（p99 97µs）写 9.2K @107µs；QD32 读 170K / 写 23K IOPS；flush 0.12s；写入全程稳定。**待做（后续）**：接口协商速率展示（IORegistry，外置盘「为什么只有 400MB/s」解释）、16GB 持续写 SLC-cliff 检测曲线、QD 扫描曲线、Stibium 式真实文件模式。

## 八、附录：竞品情报速查（2026-07 调研，供设计决策引用）

- **CleanMyMac 5**：Smart Care 五模块并行（Cleanup/Protection/Performance/Applications/My Clutter）；tile 卡 + Review；Safety Database 品牌化规则库（禁入 /System/Library，个人文件不自动选）；Moonlock 引擎（Endpoint Security framework，AV-TEST 检出 93.3%）；Performance 模块含 Free RAM/瘦身 TM 快照/维护脚本。骂点：体积虚标（purgeable 计入）、清缓存反变慢、升级压迫弹窗。
- **DaisyDisk**：把 purgeable 与 APFS 快照做成一级公民节点，可视化 + 可删 + 删后免重扫刷新 + 诚实预期（「删完还会剩 7-10GB」）——P3 直接对标。MAS 版快照功能被沙盒阉割 → Developer ID 直销路线正确性佐证。
- **Nektony**：2 万+ App 数据库 + 动态扫描混合；Remaining Files 按原 App 分组 = 孤儿检测最佳实践（P4 蓝本）。
- **Pearcleaner**：Sentinel 废纸篓监听自动弹残留（P4 蓝本）；弱点（平铺列表无分组无安全标注）= Xico 要补的差异化。
- **平台硬约束**：Photos「最近删除」10.13+ 只能 Photos 自清（展示 + 深链引导，不硬删）；purgeable 第三方无法可靠释放；快照体积估算需特权；MAS 沙盒砍清理核心能力（系统缓存/日志/语言包/登录项禁用）。
- **舆论共识**（Eclectic Light/Apple 社区）：logd 自动轮转日志无需帮忙；手动清 purgeable 无意义——Xico 文案据此建立「懂 macOS」专业心智。
