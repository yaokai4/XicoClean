## 〇、如何使用本文档

1. **按阶段执行**：P0–P7 每个阶段独立可发版，按顺序做（P1 是所有视觉阶段的底座）。

## 一、北极星与三个「签名时刻」

### 1.1 定位句（写进官网、也写进每次设计决策）

**「诚实的精密仪表」**——比 CleanMyMac 更克制诚实（不吓唬、不造假分数、删除可撤销），比 iStat/Sensei 更精致现代（原生 SwiftUI、真材质、活的图表）。彩色永远留给数据本身，情绪回报来自「看见真实的机器 + 拿回真实的空间」。

### 1.2 签名时刻（顶级产品都有一张「可截图传播的脸」）

顶级 Mac 工具的共同规律：签名时刻 = **高频动作 × 独占视觉/手势 × 情绪回报**，且必须发生在核心循环里，不在设置页。
DaisyDisk 的脸是旭日图 + 绽放动画；Things 是 Magic Plus；CleanShot 是角落悬浮卡。Xico 确立三个：

| # | 签名时刻 | 所在核心循环 | 落地阶段 |
|---|---|---|---|
| S1 | **Sunburst「绽放」钻取**——层级切换时弧的连续形变动画（DaisyDisk 为此打磨数月，是它 16 年不倒的根） | 空间透镜浏览 | P2 |
| S2 | **「收集篮」两段式删除**——待删项收进可视容器 → 非模态 5 秒倒计时 → 完成大数字 + 签名音效 + 可撤销。把 Xico 独家的「可撤销清理」从一张卡升维成一套看得见的仪式 | 每一次删除 | P2/P4 |
| S3 | **扫描 orb 收拢为结果环**——扫描完成瞬间，进度弧连续形变为结果页汇总环，配合数字 count-up 与完成音效 | 每一次扫描 | P4 |

### 1.3 声音设计（被普遍低估的高级感来源）

CleanMyMac 的高级感有一半来自音效（UX Design Awards 官方点名其「动画、声音、色彩的沉浸式氛围」；用户原话：「音效让你觉得买到了高品质软件」）。Xico 补三个签名音效，只做三个：**扫描完成**、**清理完成**、**收集篮倒计时结束**。要求：克制、短促（<600ms）、独一无二；设置页可全局关闭；跟随系统「播放用户界面声音效果」偏好。资产需外部制作（真实录音或专业合成），工程侧先接好播放管线与开关（P4）。

## 二、对标差距总表

### 2.1 空间可视化 vs DaisyDisk（$9.99 买断，3 次 App Store Best of the Year，2026-07 日区 App Store 工具付费榜 #3 · 4.8★）

Xico 的 Sunburst 底子已达「准 premium」（原创配色体系、悬停同族高亮、中心表盘、键盘/VoiceOver 全可达）。DaisyDisk 的胜负手不在图表类型，在四个打磨点 + 删除仪式：

| # | DaisyDisk 的做法 | Xico 现状 | 差距等级 |
|---|---|---|---|
| D1 | **「绽放」层级过渡动画**：钻取/返回时弧在层级间连续形变，视觉可追 | drill/pop 瞬时整树重排（SunburstView.swift:71-80），仅首现有 0.94 缩放 | ★★★ 核心 |
| D2 | 小文件**聚合成「其他」弧段**而非隐藏 | <0.5° 碎片直接丢弃（SunburstView.swift:334），用户看到的环不满 360° | ★★ |
| D3 | 扇区**按大小排序**、环宽统一、固定 5+5 环深 | 已按大小排序；maxDepth=4 可保留 | ★ 微调 |
| D4 | **导航时颜色保持**（层级可追） | 顶层分色相 + 子孙按深度减淡（SunburstView.swift:325-355），钻取后色相锚点会变 | ★★ |
| D5 | **Collector 收集器**：拖拽「摘花瓣」收集待删项，两段式删除 + 非模态 5 秒倒计时 + 删除前预览 + 实时回收空间数字 | 直接删除 + 确认；且 SunburstView 与 TreemapView 绕过 env.safety 直接实例化 XicoSafetyRules（SunburstView.swift:120-127、TreemapView.swift:52-56），与 SpaceLensModel 的 env.safety（SpaceLensView.swift:99）**两条调用口径** | ★★★ 核心 |
| D6 | 幽默文案彩蛋（「这些数据要 2.5 盒 DVD-R 才装得下」） | 无 | ★ 点缀 |

### 2.2 菜单栏 vs iStat Menus 7（$11.99 买断，行业标杆）

Xico 相对强项（勿推倒）：设置页可视化样式磁贴、模板图单色克制默认、steady-state 降采样、全传感器/风扇/每卷 SMART 面板密度、Top 进程行内占用条、快捷操作（释放内存/快速清理——iStat 没有的清理类差异化）。光栅化签名缓存目前也是强项，但 P3 渲染换血后自然退役，不必保留。

| # | iStat Menus 7 / Stats 的做法 | Xico 现状 | 差距等级 |
|---|---|---|---|
| M1 | **Combined mode**：多指标合并为单一菜单栏项，每子项仍可独立设置（刘海时代刚需、v7 招牌） | combined 项只是一枚静态 gauge 图标（MenuBarGlyph.swift:144-150），名不副实 | ★★★ 核心 |
| M2 | 直接 NSView/CALayer 自绘，零光栅化开销 | 每 tick 主线程 ImageRenderer 光栅化 N 个启用项（MenuBarController.swift:46-50 + MenuBarGlyph.swift:164-170）；CPU%/网速几乎每 tick 变 → 缓存救不了多数帧。**审计 P1 的剩余一半** | ★★★ 核心 |
| M3 | 逐屏像素精确 | scale 硬编码 =2（MenuBarGlyph.swift:166），1x 外接屏是缩放位图 | ★★ |
| M4 | 折线时间窗可调 | 固定 30 样本 ≈1 分钟（MenuBarGlyph.swift:257 + AppModel.swift:142 historyCap 60） | ★★ |
| M5 | 下拉内 hover 行项弹二级图表（三层渐进披露）；面板可钉住/独立窗口 | 面板不可钉住；XLineChart 已具备悬停擦洗能力（XLineChart.swift:31,101-125）但面板未启用（MenuPanels.swift:184,619） | ★★ |
| M6 | **基于规则的阈值通知**：CPU 超限/磁盘空间/温度/网络断连/公网 IP 变化 | 无任何阈值通知 | ★★★ |
| M7 | 每项独立刷新率（最高 0.5s）、项目可排序、每项颜色/宽度可调 | 全局共用 1/2/3s 三挡（SettingsView.swift:588-595）；顺序写死（MenuBarController.swift:25-33） | ★★ |
| M8 | 设置全部真实生效 | **僵尸设置**：border 键存在但控制器硬编码恒 true（SettingsView.swift:42-46 vs MenuBarController.swift:154）；每项 colored 控制器支持但无 UI（MenuBarController.swift:146-151）；全局 xico.mb.style 残留（SettingsView.swift:24） | ★★（信任问题） |
| M9 | 面板详情 1–2s 实时感 | 面板打开时温度/风扇 detailTick%3 → ≈6s 才刷一轮（AppModel.swift:350） | ★★ |
| M10 | 电池/蓝牙外设电量等常驻模块 | 面板有电池卡，无电池菜单栏项 | ★★ |

**差异化打法**（不追选项数量）：iStat 公认痛点 = 设置迷宫 + UI 旧；Stats 痛点 = 不精致 + 耗电（自曝传感器模块 +50% 功耗）。Xico 打「**开箱即好看**的少数精品预设 + **低功耗采样**做成可宣传卖点」。macOS 26 菜单栏默认全透明 + 用户可系统级隐藏第三方项 → 单色模板图默认、窄字形、combined 优先是面向未来的防御姿势。

### 2.3 清理体验 vs CleanMyMac 5（2024-10 大改版，$39.95+/年 或 $119.95 买断）

CleanMyMac 值得学的：Smart Care 瓦片式结果（先给结论数字，明细藏一层 Review）、声音设计、首扫 5 分钟内给出可行动成果（官方称首扫平均清出 5.5GB——这个数字本身就是转化武器）、权限延迟到模块启动时才请求。（它的「每模块渐变氛围」四件套之一 Xico 刻意不学——与「彩色留给数据本身」的北极星冲突，见第八节克制清单。）

## 三、全局设计铁律 v2（每一处都必须成立；v1 七条继续有效，此处为增补）

v1 七条（见 [xico-master-upgrade-prompt.md](./xico-master-upgrade-prompt.md) 第五节）：真实数据、边框只圈图形、黄金比间距、克制配色、一套动效、真实层次、深浅色+11语言+AA+VoiceOver+键盘。**新增：**

8. **真实材质分层**：材质只属于「导航层」（侧栏/工具条/浮层/菜单栏面板），内容层用不透明 surface；禁止 glass-on-glass 叠加；Reduce Transparency 时自动退化为磨砂/实底。macOS 26 `glassEffect` 一律走 `#available` 门控 + Material 回退。
9. **暗色高程两条腿**：暗色模式下层级区分靠「surface 随高程变亮 + 内侧 1px 高光」为主、阴影为辅（纯黑影在 0x07080F 画布上不可见）；浅色模式下白色高光是 no-op，改用暗描边/内影——所有高光/阴影必须 appearance 感知。
10. **动效物理统一**：全局标准过渡 `.spring(duration: 0.35, bounce: 0.15)`；hover/菜单等高频反馈 ≤200ms；bounce 永不超过 0.4；入场 ease-out、出场 ease-in；所有曲线只从 `XMotion` 取，禁止新增私有 spring。
11. **动画必须停表**：任何 `TimelineView(.animation)` / `repeatForever` 必须有自停条件或可见性门控；不可见即零帧。低频仪表用 `TimelineView(.periodic)`。
12. **删除的语义**：删除动作必须用 destructive 按钮变体 + 具体动词文案（「删除 23 项」而非「确定」）；**Undo 优于 Are-you-sure**——高频删除走「非模态倒计时 + 撤销」，确认弹窗只留给真正不可逆操作（粉碎）；红色 + 图标 + 文字三重标识。
13. **诚实指标**：任何「分数/评级/健康度」必须逐项可点开溯源到真实采样值；绝不出现「跑一次扫描分数就变高」的假指标（CleanMyMac 健康分的前车之鉴）。
14. **隐私即卖点**：永不上传用户数据/配置；所有统计本地存储。任何联网行为（激活/更新检查）在设置页可见可解释（Bartender 因不透明变更一夜崩盘的教训）。
15. **声音克制**：只有三个签名音效；全局可关；跟随系统 UI 音效偏好；永不在循环/hover 上加声音。

## 五、验证工作流（复用仓库既有约定）

- **菜单栏字形**：`Xico --glyphs` → `/tmp/xico-icon/*.png`（离屏，最可靠）。
- **任意页面真机**：克隆 `~/Applications/Xico.app` 到临时目录、换入 `.build/debug/Xico` + 全部 `Xico_*.bundle`、`codesign --force --deep -s -`、`--open=<moduleID>` 直达、清 `~/Library/Saved Application State/com.xico.app.savedState`、截图。
- **组件离屏**：`--layout`（注意 `ImageRenderer` 不渲染 ScrollView 内容）。
- **多语言**：`--shots --lang=<code>` 全 11 语言截断检查。
- **打包**：`scripts/make_app.sh` 必须嵌入**全部** `Xico_*.bundle`（漏一个 = `Bundle.module` 启动崩溃）。
- **能耗**：`sudo powermetrics --samplers tasks -n 12 -i 5000 | grep -i xico` 测稳态 CPU；Instruments Animation Hitches 测动画掉帧。
- **动画验收新增**：录屏逐帧检查绽放/收拢过渡的连续性（无跳帧、无中间态闪白）；Reduce Motion 打开后逐条复核降级。

**目标**：五个功能模块，全部围绕「诚实 + 常驻价值」，制造持续打开的理由（对抗清理类工具「清一次就卸载」的宿命）。

**任务**：
1. **可解释健康分**：首页健康头升级为 0-100 分 + 五个子项（磁盘余量/内存压力/温度/威胁/SMART，全部现成数据）。**每个子项可点开溯源**到真实采样值与打分公式；分数只反映机器状态，与「跑没跑扫描」无关（CleanMyMac 假健康分的反面）。权重与公式写死在 Domain、文档化。
2. **价值账本（My Activity 式）**：本地 SQLite/JSON 记录累计：清理容量、粉碎次数、拦截威胁、卸载应用数；设置页或首页入口「Xico 为你做了什么」：月度条形 + 累计大数字 + 幽默换算。纯本地、隐私叙事（「这些数据从不离开你的 Mac」）。CleanMyMac 5.4 刚上同类功能，说明头部认可其留存价值。
3. **每进程网络流量**（面板支柱内容，iStat 的护城河之一）：`nettop -P -x -L 1` 采样解析（无私有框架依赖），网络面板加 Top 4 进程双向速率行（复用 Top 进程行内条语言）。**工程风险**：nettop 输出格式跨版本波动，解析失败则该区块整体隐藏（诚实降级），不显示错误数据。
4. **云盘缓存清理**（CleanMyMac 5.2 才上的新品类，窗口还在）：扫描 iCloud Drive 可驱逐本地副本（`brctl evict`，文件保留云端）、Dropbox/Google Drive 缓存目录。**红线**：只驱逐/清缓存，永不删除云端唯一副本；UI 明确解释「文件仍在云端」。先做 iCloud（系统原生、最安全），第三方盘做只读分析报告不做删除（第一期）。
5. **威胁签名在线更新**（07 遗留 P2）：threatSignatures 从硬编码改为「内置基线 + 服务器增量更新」（复用许可证服务器通道，签名文件带版本与签名校验），设置页显示签名库版本/更新时间。

**验收**：健康分每个子项点开可见公式与原始值；账本跨重启累计正确；nettop 解析失败时区块优雅隐藏；iCloud 驱逐后文件云端可再下载实测；签名库离线时用内置基线无报错。

## 八、克制清单（明确不做——顶级感的另一半）

1. **不做**天气/日历/时钟菜单栏模块（iStat 做了且被抱怨订阅化；与「机器仪表」定位无关）。
2. **不做**恐吓式营销（红色大警告、「你的 Mac 处于危险中」）；威胁展示只陈述事实与来源。
3. **不做**激进默认勾选：任何扫描结果默认只勾「绝对安全」级（CleanMyMac 预勾 Firefox 密码数据被骂的反面）。
4. **不做**假指标：健康分与「跑没跑扫描」永远解耦。
5. **不做**数据上传：配置、统计、账本全部本地；此承诺写进「安全承诺」页（Bartender 教训：工具类护城河一半是信任）。
6. **不做**五彩纸屑式庆祝：celebration 靠「数字 + 音效 + 克制粒子」三件套，never candy, never confetti（北极星原文）。
7. **不做** glass-on-glass、内容层玻璃、装饰性 tint（Liquid Glass 采纳三原则：Content-first / Concentric / Cohesive）。
8. **不做**第三方云盘删除（第一期只读分析）；**不做** hover 打开面板做主交互（UX 文献一致反对，只可作 opt-in）。
9. **不做**订阅化本地功能：本地功能买断可用；只有真实服务器成本（签名更新/未来云能力）才可订阅（2024-2026 顶级 Mac 工具的共同定价底线：DaisyDisk $9.99 买断稳居付费榜、CleanShot 买断+1 年更新、Things 买断——订阅疲劳是真实民意）。
10. **不做**每模块渐变氛围背景（CleanMyMac 的做法）：Xico 的画布保持中性 + 两角微辉光（AppBackground 现状），彩色永远留给数据本身——模块身份靠图标 + 名称 + 语义构成条表达，不靠背景染色。
