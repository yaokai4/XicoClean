## 三、逐页对比 Sensei / iStat Menus / CleanMyMac

### 菜单栏常驻条 vs iStat Menus
- **持平或更优**：4 样式（图标+值 / 仅值 / 迷你图 / 可视化）、单色模板自适应深浅、每项独立彩色/加框、网络永不裸 0。
- **Xico 更克制**：「边框只圈图形、数值在框外」——比 iStat 的「统一背景块」更干净；框可**按项**开关，比 iStat 全局设定更细。
- **本轮新增**：图形色随主题；rich 直方图/环/条可选加框（iStat 环/条一律裸露，Xico 给了选择权）。

### 下拉面板 vs Sensei
- **CPU / 内存 / 网络**：Sensei 的频率/直方图/每核心/进程/负载/压力/分页/交换/峰值/接口——**Xico 现全部具备**，且颜色随主题统一、接口带真实 IP。
- **合并总览**：Sensei 图 3 每卡不同色调——本轮以主题指标色实现「极淡卡底 + 同色描边」，同时保留 Xico 品牌辨识。

### 监视页 vs Sensei / iStat
- 四大历史图平滑流动 + 悬停擦洗读**值·时刻** + 每核心条/环可选 + 传感器温度 + LIVE 脉冲——**持平**，主题色统一为 Xico 独有。

### 硬件页 vs Sensei（Xico 明确更全的项）
- 电池：健康环 + 循环 + 温度 + 满充/设计容量 + **功率趋势迷你图** + **健康建议**——Sensei 多为静态读数。
- 存储：SMART + TBW + 通电时长 + 读写温度 + TRIM + **寿命预警**。
- 散热：风扇当前 + **[最低,最高] 区间条**。
- 内存：容量/类型/速率/**制造商/插槽**逐条解析。

### 清理/优化 vs CleanMyMac
- 非本轮范围；既有 SafetyEngine 删除红线、扫描招牌视觉未改动，回归无影响。

## 五、「是否达到全球第一」诚实结论

**结论：在「原生 macOS 系统监控 + 硬件档案 + 可调配色」这个组合维度上，Xico 已达到 Sensei / iStat Menus 的第一梯队水准，并在若干轴上（配色统一随主题、数据真实性纪律、硬件页完整度、克制的加框美学）具备自己的独特优势。**「全球唯一第一」是一个需要长期真机口碑与生态沉淀才能盖章的称号，就单次功能/视觉横评而言，本轮已把可量化的差距基本抹平。

### 已建立的领先点
1. **单一色阶事实源**：全应用数据可视化随一套主题色阶，切主题一键全局改色——iStat 的主题多停留在菜单栏图标，App 内页面配色不总跟随。
2. **数据真实性纪律**：读不到即 `—`/隐藏，成品里无任何占位/假数据；建议类文案只从真实数值推导。
3. **硬件页完整度**：电池功率趋势 + 健康建议、存储 TBW/寿命预警、风扇区间条——逐项等于或多于 Sensei。
4. **加框美学的选择权**：只圈图形、数值在框外、按项可开关可调色。

### 剩余差距 / 后续可攻克（诚实清单）
1. **每核心「历史热力图」**：iStat 有每核心随时间的热力带；Xico 目前是每核心「当前」条/环，无逐核历史带。（需 `perCoreHistory` 已具备，可再做一层可视化。）
2. **内存带宽 / 通道数**：文档列为「可得则显」，当前 profiler 未稳定透传，**故未显示**（守「读不到不编造」优先于「凑齐字段」）。
3. **风扇目标/最大 RPM 的 SMC 目标键**：仅在 SMC 暴露目标键的机型可得；Apple Silicon 多数不透传，当前显示当前 + [最低,最高] 区间而非「目标」。
4. **显示 ProMotion / Nits**：同属「可得则显」，多数机型 IORegistry 不透传峰值亮度，未显示。
5. **传感器中心**：监视页散热卡已列 CPU/GPU/SSD/电池温 + 风扇；「硬件页 2 列传感器网格」的完整复用可再统一。
6. **rich 环/条加框的视觉权衡**：默认加框让整条菜单栏成为「仪表盘」风格；偏好极简裸露的用户可逐项关闭——这是取向而非缺陷，但默认值仍可根据真机口碑再校准。
7. **真机长跑口碑**：离屏/真实窗口快照已覆盖静态与主题变体；悬停擦洗、动效流动、长时间驻留内存/能耗仍需真机长期观察。

> 与前一轮 [`09-升级完成审计`](./09-升级完成审计-2026-07.md) 相比，本轮把「配色未统一（G1）」「加框非顶级/非按项（G2）」「样式偏文字（G3）」三项**从差距转为达成**，并补齐监视页悬停时刻、硬件健康建议、11 语言新词条。剩余差距均为「可得则显」的硬件透传边界与长期口碑，不涉及编造或半成品。

## 六、剩余差距清单的逐项攻克（R1–R7 追加轮）

> 上文 §五「剩余差距 / 后续可攻克」七项，本轮逐项处理如下。凡「可得则显」类，一律**只在硬件真实暴露该数据时显示，读不到即隐藏，绝不编造**。

| # | 差距项 | 处理 | 真实性边界 |
|---|--------|------|-----------|
| **R1** | 每核心历史热力图 | 监视页 CPU 卡新增第三视图「热力图」（当前条 / 迷你环 / **历史热力图** 三态切换）：每核心一行随时间的负载热力带（Canvas 逐列绘制，越亮越忙，>80% 叠暖色），按 IORegistry `cluster-type` 真实分 P/E 组。数据取 `engine.perCoreHistory`。 | 历史不足 2 帧时自动退回当前条，绝不画假带 |
| **R2** | 内存带宽 / 通道数 | 内存**速率**（`memorySpeed`，Intel `dimm_speed`）与**插槽/制造商**本就已解析并显示；带宽与通道数经核查 macOS **未透传**（`SPMemoryDataType` 在 Apple Silicon 仅返回容量/类型/厂商）。 | 未透传项**保持隐藏**——不做「按芯片查表」的伪读数 |
| **R3** | 风扇目标 RPM | SMC 新增读取 `F%dTg` 目标键 → `FanReading.target` → `FanInfo.target`；硬件页风扇行显示「目标 N」+ 在区间条上标一根目标刻度。 | SMC 无该键的机型（多数 Apple Silicon）**自动不显** |
| **R4** | 显示 ProMotion / Nits | ProMotion 由 `CGDisplayCopyAllDisplayModes` **真实枚举**最高刷新率推导（>60Hz 即判定），显示 ProMotion 徽标；Nits（峰值亮度）**无公开 API**，如实不显。 | 本机 M1 13"（60Hz）正确**不显** ProMotion；Nits 全平台不显 |
| **R5** | 传感器中心统一 | 监视页 CPU 标签新增**传感器中心卡**：复用硬件页的 2 列分组网格（处理器/GPU/固态/电池/环境/其他），显示全部命名温度传感器 + 归一化温度条。独立 3 秒节流采样，仅在页面可见时运行。 | 读不到传感器即**整卡隐藏** |
| **R6** | 菜单栏 rich 加框默认 | 校准为**按图形类型**取默认：图表区图形（CPU 直方图 / 网络折线）默认加框，环/条（内存·GPU·磁盘）默认裸露（对齐 Sensei 更干净），三态仍**按项可开关**。 | 取向决策已落定，非缺陷 |
| **R7** | 真机长跑口碑 | 代码侧稳健性审计通过：全部 Timer 均 invalidate 或随视图 `.onReceive` 生命周期收束；所有 Combine `.sink` 用 `[weak self]`；`MetricsEngine` 用引用计数 retain/release 平衡；新增传感器采样 3 秒节流且仅可见时运行。 | **长期内存/能耗口碑本质是观测性的**，无法在一次编码内「盖章」，需真机长跑 |

**验证**：`swift build` 绿 · `swift test` 113 项 0 失败 · `--liveshots` 新增 `21-monitor-cpu-heat`（热力图 + 传感器中心）与硬件页（风扇目标/ProMotion 徽标按真实数据显隐）· 新增 3 词条 × 11 语言 = 33 项 `plutil` 全过、覆盖 11/11。

**诚实小结**：R1/R3/R5/R6/R7 为**已落地的功能/决策**；R2/R4 的「带宽·通道数·Nits」经核查为 **macOS 不透传** 的硬边界，按「读不到即隐藏」保持不显——这是对「真实数据」铁律的坚守，而非遗漏。R7 的长跑口碑属观测范畴，代码稳健性已审计通过。
