# Apple 公证(Notarization)受限问题 · 完整解决教程

> 现状:向 Apple 提交公证时返回 **status 7000 — "Team is not yet configured for notarization"**,
> 后续提交全部卡在 "In Progress"。这是**账号层面的限制**,与代码、签名、打包方式无关。
> 本文分三部分:①彻底解决(联系 Apple 开通);②等待期间的替代分发方案;③开通后的收尾步骤。

---

## 一、为什么会这样

新注册的 Apple Developer Program 账号(尤其是个人账号/新公司账号),Apple 会对
**Developer ID 公证服务**做人工风控审核。在审核通过之前:

- `notarytool submit` 要么直接被拒(status 7000),要么永远显示 `In Progress`;
- 这不是配额、不是证书问题——本机的 Developer ID Application 证书(Team `AQ5TMWUPMH`)签名完全正常;
- **无法用技术手段绕过**,只能让 Apple 把团队标记为"可公证"。

## 二、彻底解决:联系 Apple 开通(唯一正解)

1. 登录 <https://developer.apple.com/contact/> (用购买了开发者计划的 Apple ID,即 hi@yaokai.me)。
2. 选择 **Development and Technical** → **Other Development or Technical Questions**
   (也可走 Membership and Account → Program Benefits)。
3. 建议用英文提交工单,模板:

   ```text
   Subject: Notarization service not enabled for my team (status 7000)

   Hello,

   When submitting my signed app for notarization via `xcrun notarytool submit`,
   every submission is rejected with statusCode 7000:
   "Team is not yet configured for notarization. Please contact Developer
   Programs Support."

   Team ID: AQ5TMWUPMH
   Apple ID: hi@yaokai.me
   App: Xico Clean (com.xico.app), signed with a valid
   "Developer ID Application: Kai Yao" certificate.

   Recent submission IDs (all stuck In Progress or rejected):
   - b81bdfa7-98ca-4413-a700-064eec376e41
   - 17929457-e29f-42bb-af89-2eeb188a6222

   Could you please enable notarization for my team? Thank you.
   ```

4. 也可以直接**打电话**(通常更快):中国区开发者支持 400-670-1855,工作日 9:00–17:00,
   报上 Team ID 和错误码 7000,说明需要开通 notarization。
5. 一般 1–5 个工作日内会回复/开通。开通后**无需重新打包**,直接重交历史 zip 即可。

## 三、等待期间:不受影响的分发方案

**重要认知:公证不是运行的前提,只是免除 Gatekeeper 首次打开的警告。**
未公证的 Developer ID 签名 App 完全可以分发和运行,只是用户第一次打开时多一步。

### 方案 A(当前官网已采用):教育用户右键打开
1. 用户从 mac.xicoai.com 下载 DMG,拖入"应用程序";
2. **首次打开:右键(或按住 Control 点击)Xico.app → 打开 → 再点"打开"**;
3. 之后与正常 App 无异。建议在官网下载按钮旁放一行小字说明这一步。

### 方案 B:命令行去除隔离标记(适合技术用户)
```bash
xattr -d com.apple.quarantine /Applications/Xico.app
```

### 方案 C:macOS 15+ 的新路径
系统设置 → 隐私与安全性 → 底部会出现"已阻止 Xico 打开" → 点**仍要打开**。

> 三个方案都不需要用户关闭 SIP 或改安全设置,是 Apple 官方认可的用户主动放行路径。

## 四、开通后的收尾(一次跑完)

```bash
cd /Users/yaokai/Desktop/IT/MacApp/XicoApp

# 1. 确认账号已开通:重查最近一次提交,状态应变为 Accepted
xcrun notarytool history --keychain-profile "XicoNotary" | head

# 2. 若历史提交已 Accepted:直接装订最新包并重新上传
xcrun stapler staple ~/Applications/Xico.app
# 重新打 DMG + 上传(或直接完整跑一遍发布脚本):
scripts/notarize.sh    # 构建→签名→公证(等待)→装订→DMG→公证 DMG→appcast

# 3. 上传新 DMG 到服务器(装订后的包离线也能过 Gatekeeper)
scp -i ~/Desktop/IT/web/Shangence.pem dist/Xico-Clean.dmg ec2-user@52.198.247.142:/tmp/
ssh -i ~/Desktop/IT/web/Shangence.pem ec2-user@52.198.247.142 \
  "sudo docker cp /tmp/Xico-Clean.dmg xicoai-app:/app/uploads/Xico-Clean.dmg && \
   sudo docker exec -u root xicoai-app chown nextjs:nodejs /app/uploads/Xico-Clean.dmg"

# 4. 验证:任何一台 Mac 下载后直接双击应无警告
spctl -a -vv /Applications/Xico.app   # 应显示 accepted · Notarized Developer ID
```

## 五、常见误区(不要做)

- ❌ **不要**教用户 `sudo spctl --master-disable`(全局关闭 Gatekeeper)——危险且 Apple 明令禁止引导;
- ❌ **不要**换免费 Apple ID 自签分发——免费账号连 Developer ID 证书都没有,只会更糟;
- ❌ **不要**反复重交公证——账号没开通前交多少次都一样,等支持回复即可;
- ❌ 卡住的 "In Progress" 提交**不占配额、无需取消**,开通后它们会自动出结果。
